# SPA Architecture: TanStack Router + Query + Electrobun RPC

## Critical Constraint: Hash Routing Required

`views://` is a filesystem-backed URL scheme — no server-side fallback exists. HTML5 `pushState` to `views://app/settings` would resolve to a nonexistent file and fail. **Hash routing is mandatory** in production (`views://app/index.html#/settings`). TanStack Router supports this via `createHashHistory()`.

## Architecture Overview

Single-page app at `views://app/index.html`. TanStack Router handles client-side navigation. TanStack Query manages server state with RPC as the transport layer. All "backend" calls go through Electrobun's typed RPC — no HTTP, no network dependency.

```
┌─────────────────────────────────────────────────────────┐
│  Bun Process (src/bun/)                                 │
│  ┌─────────────────────────────────────────────────────┐│
│  │ RPC Handlers (src/bun/rpc/)                         ││
│  │  requests: getData, createItem, ...                 ││
│  │  messages: logToBun, ...                            ││
│  └─────────────────────────────────────────────────────┘│
└──────────────────────┬──────────────────────────────────┘
                       │ Electrobun Typed RPC (IPC)
┌──────────────────────▼──────────────────────────────────┐
│  Browser View (src/app/)                                │
│  ┌───────────────┐  ┌────────────────────────────────┐  │
│  │ TanStack Query │  │ TanStack Router (hash history) │  │
│  │ queryFn → RPC  │  │ file-based routes (flat)       │  │
│  └───────┬───────┘  └────────────────────────────────┘  │
│          │                                               │
│  ┌───────▼──────────────────────────────────────────┐   │
│  │ Electroview singleton (src/lib/electroview.ts)    │   │
│  │ electroview.rpc.request.xxx()                     │   │
│  └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### `src/app/index.tsx`

App entry. Creates QueryClient, router, and mounts the app.

```tsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { createHashHistory, createRouter, RouterProvider } from "@tanstack/react-router";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { routeTree } from "./routeTree.gen";
import "@/styles/global.css";
import "@/lib/electroview"; // Initialize Electroview early (enables drag regions)

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      networkMode: "always", // RPC is local IPC — always available
      staleTime: 1000 * 60 * 5,
    },
    mutations: {
      networkMode: "always",
    },
  },
});

const hashHistory = createHashHistory();

const router = createRouter({
  routeTree,
  context: { queryClient },
  history: hashHistory,
});

declare module "@tanstack/react-router" {
  interface Register {
    router: typeof router;
  }
}

createRoot(document.getElementById("root")!).render(
  <StrictMode>
    <QueryClientProvider client={queryClient}>
      <RouterProvider router={router} />
    </QueryClientProvider>
  </StrictMode>,
);
```

### `src/bun/index.ts`

Main process. Wires RPC to the window.

```ts
import { BrowserWindow, Updater } from "electrobun/bun";
import { rpc } from "./rpc";

const DEV_SERVER_PORT = 5173;
const DEV_SERVER_URL = `http://localhost:${DEV_SERVER_PORT}`;

async function getAppUrl(): Promise<string> {
  const channel = await Updater.localInfo.channel();
  if (channel === "dev") {
    try {
      await fetch(DEV_SERVER_URL, { method: "HEAD" });
      console.log(`HMR enabled: Using Vite dev server at ${DEV_SERVER_URL}`);
      return DEV_SERVER_URL;
    } catch {
      console.log("Vite dev server not running. Run 'bun run dev:hmr' for HMR support.");
    }
  }
  return "views://app/index.html";
}

const url = await getAppUrl();

const _mainWindow = new BrowserWindow({
  title: "Electrobun-Shadcn",
  url,
  titleBarStyle: "hiddenInset",
  transparent: true,
  styleMask: { borderless: false },
  rpc,
});

console.log("Electrobun Shadcn app started!");
```

### Query/Mutation Hook Pattern

```ts
// src/hooks/queries/use-items.ts
import { useQuery } from "@tanstack/react-query";
import { electroview } from "@/lib/electroview";

export function useItems(filter?: string) {
  return useQuery({
    queryKey: ["items", { filter }],
    queryFn: () => electroview.rpc.request.getItems({ filter }),
  });
}

// src/hooks/mutations/use-create-item.ts
import { useMutation, useQueryClient } from "@tanstack/react-query";
import { electroview } from "@/lib/electroview";

export function useCreateItem() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: (data: { name: string }) => electroview.rpc.request.createItem(data),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ["items"] });
    },
  });
}
```

## TanStack Query: Why `networkMode: "always"`

Electrobun RPC is local IPC between the Bun process and the webview. There is no HTTP, no network socket, no possibility of "offline." TanStack Query's default `networkMode: "online"` would pause queries when the OS reports no network — which is wrong for RPC. Setting `networkMode: "always"` globally ensures queries always execute regardless of OS network state.

For persistent cache across app restarts, add `@tanstack/query-persist-client-core` later. Not needed for initial setup.

## File-Based Routing: Flat Convention

Using flat file naming (`.` separator for nesting, no subdirectories per route):

```
src/app/routes/
  __root.tsx                    ← Root layout (always rendered)
  index.tsx                     ← /
  auth.tsx                      ← /auth
  settings.tsx                  ← /settings (layout with <Outlet />)
  settings.profile.tsx          ← /settings/profile
  settings.notifications.tsx    ← /settings/notifications
  posts.$postId.tsx             ← /posts/:postId (dynamic)
```

Co-located non-route files use `-` prefix to be ignored:

```
  -components/                  ← Ignored by router plugin
    post-card.tsx
```

The `routeTree.gen.ts` file is auto-generated. Add to `.gitignore` or treat as generated artifact.

## Migration Steps

### Phase 1: Install Dependencies + Config

1. Install TanStack packages
2. Update `vite.config.ts` (root, plugins, remove rollupOptions)
3. Update `electrobun.config.ts` (copy rule)
4. Add `routeTree.gen.ts` to ignore files (oxlint, oxfmt, git)

### Phase 2: Create SPA Structure

1. Create `src/app/index.html`
2. Create `src/app/index.tsx` (with router + query setup)
3. Create `src/app/routes/__root.tsx` (layout from current `LayoutSidebar`)
4. Create `src/app/routes/index.tsx` (dashboard from current `Page.tsx`)
5. Create `src/app/routes/auth.tsx` (placeholder)

### Phase 3: RPC Scaffold

1. Create `src/lib/types/rpc.ts`
2. Create `src/lib/electroview.ts`
3. Create `src/bun/rpc/index.ts`
4. Update `src/bun/index.ts` (wire RPC, update URLs)
5. Create `src/hooks/queries/` and `src/hooks/mutations/` with example hooks

### Phase 4: Cleanup

1. Delete `src/mainview/` entirely
2. Update `LayoutSidebar` — remove wrapper role (root route handles it now)
   or delete if fully absorbed into `__root.tsx`
3. Verify `@source` paths in `global.css` still cover `src/app/routes/`
4. Run `vite build`, verify `dist/` output
5. Run `electrobun dev`, verify app loads at `views://app/index.html`

## Adding a New Route

1. Create `src/app/routes/my-page.tsx`
2. Export `Route = createFileRoute("/my-page")({ component: MyPage })`
3. Done. TanStack Router plugin auto-regenerates the route tree.

## Adding a New RPC Endpoint

1. Add type to `src/lib/types/rpc.ts` (both `bun.requests` and optionally `webview.requests`)
2. Add handler in `src/bun/rpc/index.ts` (or a handler module)
3. Create query hook in `src/hooks/queries/` that calls `electroview.rpc.request.xxx()`
4. Use hook in route component
